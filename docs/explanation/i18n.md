# Internationalization (i18n)

OpenClinic is designed with a multi-language architecture, allowing the interface to adapt to different user languages automatically.

## Supported Languages

Currently, the following languages are fully supported and translated:

* **English** (`en`) - Default
* **Spanish** (`es`) - Spanish (Spain)
* **French** (`fr`) - French
* **Portuguese** (`pt`) - Portuguese
* **Swahili** (`sw`) - Swahili

## Automatic Language Detection

The application uses Django's `LocaleMiddleware` to determine the user's preferred language. The system follows this priority order:

1. **URL Prefix**: If the URL includes a language code (e.g., `/es/`), that language is used.
2. **Session**: If the user has explicitly selected a language in their current session.
3. **Cookie**: If the user has a `django_language` cookie set.
4. **Browser Settings**: If none of the above are set, the system looks at the **`Accept-Language` HTTP header** sent by the web browser. The application will match the most preferred browser language with the closest available translation in OpenClinic.
5. **Default**: If no match is found, it falls back to English (`LANGUAGE_CODE` setting).

## Technical Implementation

### Translation Files

Translations are stored in `.po` (Portable Object) files under the `locale/` directory:

* `locale/es_ES/LC_MESSAGES/django.po`
* `locale/fr_FR/LC_MESSAGES/django.po`
* `locale/pt/LC_MESSAGES/django.po`
* `locale/sw/LC_MESSAGES/django.po`

### Compilation

To be used by the application, these human-readable files must be compiled into binary `.mo` files. This is handled automatically by the project's orchestration:

* **Development**: Running `make dev` automatically triggers the compilation of all languages.
* **Docker**: The `Dockerfile` includes a build step that compiles the messages, ensuring they are available in production images.

If you need to compile them manually, you can run:

```bash
make compilemessages
```

### Adding New Languages

To add a new language, follow these steps:

1. Add the language code to the `LANGUAGES` list in `openclinic/settings/base.py`.
2. Generate the translation files:

   ```bash
   python manage.py makemessages -l <language_code>
   ```

3. Translate the strings in `locale/<language_code>/LC_MESSAGES/django.po`.
4. Compile the messages:

   ```bash
   make compilemessages
   ```
